# Signal — база знаний проекта

Этот репозиторий пока не содержит программного кода. Он используется как рабочая база для проверки идеи, выбора продуктового направления и подготовки реализации Signal.

Дата актуализации анализа: **21 июля 2026 года**.

## Рабочий landing

Первая реализованная поверхность Signal находится в [`apps/landing`](./apps/landing/).

```bash
cd apps/landing
npm install
npm run dev
```

Локальный адрес: `http://localhost:3000`.

Landing включает 520vh scroll-linked product reveal с собранным и exploded Signal Core, Canvas-трансформацией MIDI в четыре направления, интерактивный Piano Roll demo, переключаемые MIDI-кандидаты, звуковой preview по явному действию пользователя, FL Studio workflow, Creator DNA, privacy-секцию, адаптивную версию и поддержку `prefers-reduced-motion`.

## Рекомендуемая формулировка продукта

**Signal — персональный MIDI-copilot внутри DAW, который понимает музыкальный контекст и творческие предпочтения пользователя, предлагает редактируемые варианты и никогда не забирает у музыканта контроль.**

Первый целевой хост — FL Studio. Первый результат — не готовая песня и не WAV, а редактируемые ноты в Piano Roll.

## Главный вывод анализа

Исходная идея сильна в философии «tools, not replacements» и в персонализации по музыкальному вкусу. Но первая версия, построенная вокруг загрузки пяти чужих песен и внешней генерации аудио, одновременно создаёт слишком много рисков: авторские права, тяжёлый аудиоанализ, слабая управляемость результата, стоимость inference и неудобный перенос результата в DAW.

Более сильный путь:

1. Начать с выделенного пользователем MIDI в FL Studio.
2. Делать операции `Continue`, `Variation`, `Harmonize`, `Bassline`, `Simplify`, `Humanize`.
3. Вставлять результат прямо в Piano Roll как редактируемые ноты.
4. Накапливать только согласованные сигналы обратной связи: что сохранено, изменено или отклонено.
5. Затем построить персональный **Creator DNA** сначала по собственным MIDI-проектам пользователя.
6. Только после подтверждения ценности добавить анализ референсных аудиотреков.
7. После FL-версии вынести ядро в VST3/CLAP и другие DAW.

## Правило работы с документацией

- Факты сопровождаются источниками.
- Непроверенные утверждения помечаются как гипотезы.
- Решения меняются только вместе с причиной и датой.
- Каждый этап разработки имеет измеримый gate: продолжить, изменить или остановить.
