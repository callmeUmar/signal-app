# 04. Этапы, аналитика и критерии запуска

## Phase L0 — Message prototype, 3–5 дней

- hero copy variants;
- low-fi page;
- before/after Piano Roll animation prototype;
- 5-second comprehension test с 10 целевыми users;
- primary CTA test.

Gate:

- ≥80% понимают, что output — editable MIDI;
- ≥70% понимают contextual workflow;
- browser и FL routes не путаются.

## Phase L1 — Foundation, 1 неделя

- project setup;
- shared design tokens;
- responsive navigation/footer;
- homepage sections;
- privacy/terms placeholders linked to approved documents;
- analytics events;
- preview deployment.

## Phase L2 — Browser demo, 1 неделя

- precomputed/deterministic MIDI example;
- playback;
- candidate comparison;
- visual diff;
- CTA into dashboard onboarding;
- accessibility/reduced motion.

## Phase L3 — FL beta funnel, 1 неделя

- `/fl-studio`;
- browser auth walkthrough;
- waitlist/invite state;
- `/download` release manifest integration;
- supported versions;
- docs/troubleshooting links.

## Phase L4 — Public readiness, 3–5 дней

- content/legal review;
- cross-browser/mobile testing;
- performance/accessibility audit;
- analytics validation;
- broken link scan;
- security headers;
- incident/degraded states;
- rollback.

## Acceptance criteria

### Product

- hero describes concrete action;
- CTA `Try in browser` works end-to-end;
- sidebar state is accurate: waitlist/beta/stable;
- no unbuilt feature presented as available;
- browser and sidebar relationship is clear;
- export/control/privacy visible before signup.

### Technical

- no client secrets/tokens in source or URLs;
- auth redirects allowlisted;
- forms protected;
- download artifact/checksum generated by release pipeline;
- app remains informative during backend outage;
- accessibility keyboard path complete;
- no autoplay audio;
- analytics contain no creative content.

### Measurement

- visitor → demo start;
- demo start → candidate playback;
- demo completion → signup;
- FL page → beta join/download;
- signup → first dashboard project;
- acquisition source preserved without leaking auth state.

## Launch blockers

- misleading native-sidebar claim;
- fake testimonials;
- broken privacy/terms links;
- unsigned or mismatched download;
- token in URL/log;
- demo loses user work without warning;
- mobile page blocked by large DAW animation;
- no supported-version information.
